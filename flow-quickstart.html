Quick Start

Five minutes from install
to flow.

This is the fastest path through the FLOW Method. Install the plugin, run one command, watch chaos become a structured project, then settle into the daily rhythm. No prior setup. No knowing what your "stack" is.

Click through the phases below and look at the dump examples to see what the method feels like before you start.

Step 01

Install the plugin.

FLOW runs inside Claude Code or CoWork. The plugin file is a single .plugin archive — installation takes about 30 seconds.

1
Download flow.plugin from the FLOW site or the project repo. Save it anywhere you can find it.
2
Open Claude Code or CoWork. Find the plugin panel (Settings → Plugins, or the plugin icon in the sidebar).
3
Drag flow.plugin into the panel. Click Save plugin. It validates and installs in a few seconds.
4
Open a new chat. Type /spinup — if the command autocompletes, you are installed.
Step 02

Pick your starting point.

FLOW gives you three ways to start a project. Pick the one that matches what you actually have — not what you think you should have.

Path A · Default
You have chaos.
Scattered inputs. Half-formed idea. You don't yet know the stack, the deadline, or even the name.
Open an inbox and dump whatever you have. Repeat across as many sessions as you want. FLOW listens without asking questions.
/spinup
Path B
You have an idea, but it's unstructured.
You can talk about it in prose, just not in bullet points.
Brain-dump the whole idea in one shot. FLOW listens, then distills it into a clean product-brief.md.
/project-strategy
Path C
You know the shape.
Name, stack, deadline — all in your head. You want the scaffold right now.
Structured Q&A. FLOW asks four questions and scaffolds immediately.
/new-project
Step 03

The four phases (click through).

Path A — chaos — is the FLOW Method default. The methodology runs in four phases: chaos, spinup, crystallize, flow. Click each card on the left to see what to do in that phase.

Phase 01 · Chaos

You arrive scattered.

The idea is half-formed. Screenshots, a half-baked Notion doc, a few links, voice notes, references. You don't know enough yet to answer "what stack" or "what deadline." Most methodologies want you to know. FLOW says don't pretend you do.

  • You have an itch for a product but no plan
  • Inputs are scattered across browser tabs, Notes, Slack DMs
  • You've been thinking about it for weeks, never sat down
  • You can describe it differently each time you mention it
Do this: open Claude. Type /spinup. That moves you to phase 02.
Phase 02 · Spinup

Open an inbox and dump.

Run /spinup and FLOW opens a chaos inbox in your current folder. Drop anything — text, files, screenshots, URLs. Repeat as many times as you want, across as many sessions as you want. FLOW captures verbatim, never asks questions, and writes a one-line tally each time so you know what it heard.

  • Type free text and hit send. That's a dump.
  • Attach a wireframe screenshot, a competitor URL, a draft doc
  • Come back tomorrow and run /spinup again with more thoughts
  • Use /spinup show to peek at the running picture without adding
What you do NOT do: answer questions, fill in fields, name the project, pick a stack. FLOW will surface those at crystallize time, not now.
Phase 03 · Crystallize

Structure on your schedule.

When the picture feels clear enough, run /crystallize. FLOW reads the inbox, surfaces contradictions and inferred fields, asks for the blocking gaps (project name, audience or problem), then scaffolds the full project: brief, hub JSON, dashboard HTML, CLAUDE.md, folder structure. The raw chaos archives into the project for permanent reference.

  • You decide when ready — could be one dump, could be fifteen
  • FLOW asks for the project name if it could not infer one
  • Eight artifacts get created atomically — all or nothing
  • Use /crystallize dry-run to preview what would be created
The chaos is preserved. The original inbox moves to _archive/kickoff-inbox/ inside the new project folder. You can always go back to the raw dumps.
Phase 04 · Flow

Daily work, daily rituals.

The project is real. The hub is your single source of truth. Six skills handle the day-to-day on natural language — no forms, no spreadsheets. The methodology stops being something you think about and starts being how you work.

  • morning kickoff reads the hub and proposes today's top three
  • note: anything routes mid-session captures to the right place
  • session close logs everything and syncs the dashboard
  • evening research queues research for tomorrow's kickoff
  • doc sync cascades decisions through tracked docs
  • generate gantt builds a timeline from your strategy docs
You stay in flow. No system maintenance time. The rituals fit between the work, not on top of it.

Step 05

What /crystallize builds for you.

Eight artifacts. Atomic — all created together, never partial. The raw chaos archives in alongside.

01
docs/product-brief.md
The brief, written in your own words, in a template adapted to your project type.
02
PROJECT_HUB.json
Single source of truth — sessions, backlog, decisions, bugs, rules, integrations, Gantt.
03
project-hub.html
Live dashboard that reads the hub JSON. Dusk + morning themes. Serve over HTTP.
04
index.html
Redirect so the bare URL serves the dashboard. No more typing /project-hub.html.
05
CLAUDE.md
Agent context. Every agent on this project reads it first. Project-specific.
06
SESSION_NOTES.md
Mid-session staging for rules, decisions, bugs. Swept into the hub at session close.
07
docs/ specs/ logs/
Empty scaffold folders. Drop product docs, agent task specs, and end-of-session logs as you work.
08
_archive/kickoff-inbox/
Your raw chaos preserved permanently — every dump, every attachment, every link.
Cheat sheet

Every command, every trigger.

All the commands and natural-language triggers FLOW listens for. Print this. Keep it on your second monitor.

Command What it does When to use it
/spinup Opens a chaos inbox in the current folder. Drops anything you give it. Re-extracts the picture each time. You're in chaos — scattered inputs, half-formed idea.
/spinup show Prints the current STATUS.md without adding a dump. You want to peek at what FLOW has extracted so far.
/crystallize Reads the inbox, scaffolds 8 artifacts, archives the chaos. When the inbox picture is clear enough to scaffold.
/crystallize dry-run Prints what would be created. Writes nothing. You want to preview the scaffold before committing.
/project-strategy Brain-dump in one shot. FLOW distills into a clean brief. You can articulate the project in prose but not in bullets.
/new-project Structured Q&A. Scaffolds the project immediately. You know the name, stack, deadline, and want the scaffold now.
"morning kickoff" Reads the hub. Surfaces priorities. Proposes top-3 plan. Start of any work session on an existing project.
"note: anything" Captures rules, bugs, decisions, tasks, research. Routes each to the right place. Mid-session — anytime something matters that would otherwise get lost.
"session close" · "quick close" Updates the hub, logs the session, syncs the dashboard, sweeps SESSION_NOTES.md. End of every work session.
"evening research" Searches, synthesizes, writes NEXT_SESSION.md for tomorrow's kickoff. When you flagged something to look into before tomorrow.
"sync docs" Cascades a decision through all tracked product/strategy docs. After a decision lands that affects the brief, PRD, or tech spec.
"generate Gantt" Builds a phased timeline from your strategy docs and your deadline. After your brief and PRD are filled in.
